# 账务核心系统流程图索引

本目录包含账务核心系统的所有业务流程图，使用 Mermaid 格式编写。

## 📋 流程图清单

### 核心业务流程

#### 1. **标准入账全流程详细图** (`standard_posting_flow_detailed.mmd`)
**用途**：详细展示标准入账的四个阶段完整流程

**包含内容**：
- 阶段一：记账流水（幂等校验、持久化、会计日期确定）
- 阶段二：凭证生成（规则匹配、借贷平衡校验、辅助核算项分摊、缓冲规则匹配）
- 阶段三：事务处理（事务创建、凭证关联、编程式事务）
- 阶段四：过账更新（单边记账判断、锁控制、余额计算、Pre/Post 快照、分录状态更新、凭证状态检查）
- MQ 异步处理流程（MQ 消费、异步过账、重试机制、失败回滚）

**关键决策点**：
- 幂等校验
- 借贷平衡校验
- 缓冲规则匹配
- 单边记账判断（is_unilateral=1 实时，is_unilateral=0 异步）
- 余额充足校验
- 凭证状态检查（所有分录都已过账才更新凭证为已过账）

**MQ 异步处理说明**：
- 异步分录通过 MQ 消息队列处理
- 消费端独立事务，执行相同的过账逻辑（锁控制、余额计算、明细记录）
- 更新分录状态为 status=2（已过账）
- 检查凭证的所有分录是否都已过账，如果是则更新凭证 status=3（已过账）
- 失败重试机制：最多重试 3 次，超过则触发回滚流程
- 异步分录失败回滚：参考事务回滚流程图的"异步分录失败回滚"场景

#### 2. **缓冲记账三种模式流程图** (`buffer_posting_modes.mmd`)
**用途**：详细展示三种缓冲模式的处理逻辑

**包含内容**：
- 缓冲规则匹配与写入
- 模式 1：逐条缓冲（实时触发，准实时入账）
- 模式 2：日间批量（定时触发，如每 5 分钟）
- 模式 3：日终批量（日切触发，对实时性要求最低）
- 锁升级策略（乐观锁 → 重试 → 悲观锁）
- Running Balance 计算逻辑

**关键决策点**：
- 缓冲模式选择
- 锁升级策略
- Running Balance 计算

#### 3. **红冲流程图** (`reversal_flow.mmd`)
**用途**：详细展示红冲的完整流程

**包含内容**：
- 前置校验（原凭证状态、分录过账、重复红冲）
- 生成红冲凭证（方向对调、金额保持正数）
- 执行红冲过账（更新账户余额、生成明细）
- 更新原凭证状态（status=4 已冲销）
- 红冲后的科目总账统计

**关键决策点**：
- 原凭证状态校验
- 分录过账校验
- 重复红冲校验

#### 4. **逻辑日切五阶段流程图** (`eod_five_phases.mmd`)
**用途**：详细展示日切的五个阶段

- 阶段 4：试算平衡（生成 `t_account_balance`、借贷平衡校验、总分核对）
- 阶段 5：归档（标记日切完成、生成报告）

**关键决策点**：
- 存量清理完成校验
- 借贷平衡校验
- 总分核对校验
- 校验失败处理

#### 7. **事务回滚流程图** (`transaction_rollback_flow.mmd`)
**用途**：详细展示事务回滚的完整处理逻辑

**包含内容**：
- 回滚触发场景（余额不足、借贷不平衡、账户状态异常、系统异常、业务规则校验失败）
- 事务回滚处理流程（捕获异常、自动回滚、更新事务状态、记录日志、补偿处理、触发告警）
- 回滚后数据状态（业务流水、事务、凭证、账户、明细）
- 特殊场景：异步分录失败回滚（单边记账场景，不生成新凭证，直接反向操作）
- 重试机制（临时性异常 vs 永久性异常）

**关键决策点**：
- 是否已创建事务
- 是否需要补偿处理
- 异常类型是否可重试

**重要说明**：
- 编程式事务内的失败会自动回滚，不需要生成红冲凭证
- 异步分录失败（跨事务）时：
  - 不生成新凭证（避免借贷不平衡）
  - 直接反向操作：更新分录状态为失败、反向更新账户余额、记录反向明细
  - 通过状态管理控制流程，通过明细表保留审计追溯

#### 8. **记账自动开户流程图** (`auto_account_opening_flow.mmd`)
**用途**：详细展示记账时自动开户的完整流程

**包含内容**：
- 账户存在性检查（解析记账规则，获取涉及的账户列表）
- 自动开户流程（匹配开户模板、检查自动开户标识、执行开户）
- 异常处理（模板不存在、模板不支持自动开户、开户失败）
- 继续记账流程（所有账户准备就绪后继续标准入账流程）

**关键决策点**：
- 账户是否存在
- 开户模板是否存在
- 模板是否支持自动开户（`auto_open=1`）
- 开户是否成功

**应用场景**：
- 解决已经在线上运行的业务，没有事先开户的问题
- 新客户首次交易时自动开户

**重要说明**：
- 自动开户在"阶段二：凭证生成"之前执行
- 开户失败会导致记账失败
- 开户成功后会记录开户日志，便于审计

---

#### 9. **期末结转流程图** (`period_end_transfer_flow.mmd`)
**用途**：详细展示期末结转的完整流程

**包含内容**：
- 前置校验（检查会计期间是否已关账、是否已执行期末结转）
- 加载结转规则（查询启用的规则，按执行顺序排序）
- 执行结转（遍历规则、查询源科目余额、计算结转金额、生成结转凭证、执行入账）
- 结转后处理（检查执行结果、标记结转完成或触发告警）

**关键决策点**：
- 当前会计期间是否已关账
- 是否已执行期末结转
- 是否有可执行规则
- 源科目是否有余额
- 入账是否成功
- 是否全部执行成功

**结转类型**：
- 损益结转：将损益类科目（收入、费用）余额结转到本年利润科目
- 成本结转：将成本类科目余额结转到相关科目
- 自定义结转：根据业务需求自定义结转规则

**重要说明**：
- 期末结转在日切流程的"阶段 5：归档"之前执行
- 支持通配符匹配源科目（如 6* 匹配所有 6 开头的科目）
- 结转凭证通过标准入账流程处理，确保借贷平衡
- 部分规则失败时触发告警，需要人工介入

## 🔗 流程图关系

```
标准入账全流程
    ├── 记账自动开户流程（阶段二之前：账户存在性检查）
    ├── 缓冲记账三种模式（阶段二：缓冲规则匹配）
    ├── 红冲流程（特殊入账场景）
    ├── 冻结/解冻流程（特殊入账场景）
    └── 事务回滚流程（异常处理场景）

逻辑日切五阶段
    ├── 存量清理 → 缓冲记账三种模式（模式 3）
    ├── 试算平衡 → 标准入账全流程（数据来源）
    └── 归档前 → 期末结转流程（可选）

账户开户流程
    ├── 标准入账全流程（开户后的记账）
    └── 记账自动开户流程（记账时自动开户）

期末结转流程
    └── 标准入账全流程（结转凭证的入账）
```

## 📊 使用建议

### 开发阶段
1. **需求理解**：先阅读 `standard_posting_flow_detailed.mmd` 理解核心入账流程
2. **特殊场景**：根据具体需求阅读对应的专项流程图
3. **日终处理**：重点关注 `eod_five_phases.mmd` 的五个阶段

### 代码实现
1. **按流程图实现**：严格按照流程图的决策点和处理步骤编写代码
2. **异常处理**：关注流程图中的错误分支，确保异常处理完整
3. **状态机**：根据流程图中的状态转换实现状态机逻辑

### 测试验证
1. **单元测试**：覆盖流程图中的所有决策点
2. **集成测试**：验证完整流程的端到端执行
3. **异常测试**：验证所有错误分支的处理逻辑
