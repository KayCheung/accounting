# 账务核心系统流程图索引

本目录包含账务核心系统的所有业务流程图，使用 Mermaid 格式编写。

## 📋 流程图清单

### 核心业务流程

#### 1. **标准入账全流程详细图** (`standard_posting_flow_detailed.mmd`)
**用途**：详细展示标准入账的四个阶段完整流程

**包含内容**：
- 阶段一：记账流水（幂等校验、持久化、会计日期确定）
- 阶段二：凭证生成（规则匹配、借贷平衡校验、辅助核算项分摊、缓冲规则匹配）
- 阶段三：事务处理（事务创建、凭证关联、编程式事务）
- 阶段四：过账更新（单边记账判断、锁控制、余额计算、Pre/Post 快照）

**关键决策点**：
- 幂等校验
- 借贷平衡校验
- 缓冲规则匹配
- 单边记账判断
- 余额充足校验

---

#### 2. **缓冲记账三种模式流程图** (`buffer_posting_modes.mmd`)
**用途**：详细展示三种缓冲模式的处理逻辑

**包含内容**：
- 缓冲规则匹配与写入
- 模式 1：逐条缓冲（实时触发，准实时入账）
- 模式 2：日间批量（定时触发，如每 5 分钟）
- 模式 3：日终批量（日切触发，对实时性要求最低）
- 锁升级策略（乐观锁 → 重试 → 悲观锁）
- Running Balance 计算逻辑

**关键决策点**：
- 缓冲模式选择
- 锁升级策略
- Running Balance 计算

---

#### 3. **红冲流程图** (`reversal_flow.mmd`)
**用途**：详细展示红冲的完整流程

**包含内容**：
- 前置校验（原凭证状态、分录过账、重复红冲）
- 生成红冲凭证（方向对调、金额保持正数）
- 执行红冲过账（更新账户余额、生成明细）
- 更新原凭证状态（status=4 已冲销）
- 红冲后的科目总账统计

**关键决策点**：
- 原凭证状态校验
- 分录过账校验
- 重复红冲校验

---

#### 4. **逻辑日切五阶段流程图** (`eod_five_phases.mmd`)
**用途**：详细展示日切的五个阶段

**包含内容**：
- 阶段 1：瞬间切日（更新全局会计日期）
- 阶段 2：存量清理（处理缓冲明细、处理未完成事务）
- 阶段 3：余额快照（保存到 `t_account_balance_snapshot`）
- 阶段 4：试算平衡（生成 `t_account_balance`、借贷平衡校验、总分核对）
- 阶段 5：归档（标记日切完成、生成报告）

**关键决策点**：
- 存量清理完成校验
- 借贷平衡校验
- 总分核对校验
- 校验失败处理

---

#### 5. **冻结/解冻流程图** (`freeze_unfreeze_flow.mmd`)
**用途**：详细展示冻结和解冻的处理逻辑

**包含内容**：
- 冻结流程（主账户不变、可用减少、冻结增加）
- 解冻流程（主账户不变、冻结减少、可用增加）
- 超时自动解冻（兜底任务）
- 冻结扣款（直接在冻结额度上扣款）
- 余额关系示例

**关键决策点**：
- 冻结有效期判断
- 超时自动解冻触发

---

#### 6. **账户开户流程图** (`account_opening_flow.mmd`)
**用途**：详细展示开户的完整流程

**包含内容**：
- 外部客户账户开户（基于模板）
- 内部账户开户（基于科目配置）
- 子账户自动创建（可用/冻结）
- 账户编号生成规则
- 开户校验

**关键决策点**：
- 模板存在且启用校验
- 科目末级校验
- 账户重复校验

---

## 🔗 流程图关系

```
标准入账全流程
    ├── 缓冲记账三种模式（阶段二：缓冲规则匹配）
    ├── 红冲流程（特殊入账场景）
    └── 冻结/解冻流程（特殊入账场景）

逻辑日切五阶段
    ├── 存量清理 → 缓冲记账三种模式（模式 3）
    └── 试算平衡 → 标准入账全流程（数据来源）

账户开户流程
    └── 标准入账全流程（开户后的记账）
```

## 📊 使用建议

### 开发阶段
1. **需求理解**：先阅读 `standard_posting_flow_detailed.mmd` 理解核心入账流程
2. **特殊场景**：根据具体需求阅读对应的专项流程图
3. **日终处理**：重点关注 `eod_five_phases.mmd` 的五个阶段

### 代码实现
1. **按流程图实现**：严格按照流程图的决策点和处理步骤编写代码
2. **异常处理**：关注流程图中的错误分支，确保异常处理完整
3. **状态机**：根据流程图中的状态转换实现状态机逻辑

### 测试验证
1. **单元测试**：覆盖流程图中的所有决策点
2. **集成测试**：验证完整流程的端到端执行
3. **异常测试**：验证所有错误分支的处理逻辑
