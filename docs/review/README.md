# docs/review · Review 提示词说明

> 本目录存放 TL 进行 Code Review 和提示词 Review 时使用的提示词文件。

## 文件清单

| 文件 | 用途 | 使用时机 |
|------|------|----------|
| `code-review.md` | Java 代码 Review | 成员提交 PR 或上传代码文件时 |
| `prompt-review.md` | Step 文件 / 四段式 Prompt Review | 成员填写 Step 文件或发起任务前 |

## 快速使用

### 代码 Review
1. 打开新会话
2. 将 `code-review.md` 全文作为系统提示词粘贴
3. 粘贴需要 Review 的代码，或上传文件
4. AI 输出：总结 + P0/P1/P2 问题列表 + 通过项

### 提示词 Review
1. 打开新会话
2. 将 `prompt-review.md` 全文作为系统提示词粘贴
3. 粘贴需要 Review 的 Step 文件内容或四段式 Prompt
4. AI 输出：总结 + 必须修正项 + 建议优化项 + 通过项

## 问题严重程度说明

| 级别 | 含义 | 处理方式 |
|------|------|----------|
| P0 | 财务规范 / 事务锁 / 架构隔离违反 | 必须修复，不得合并 |
| P1 | 编码规范严重违反（BigDecimal / 分层等） | 需修复后合并 |
| P2 | 代码质量建议（注释 / 枚举 / 命名等） | 建议修复，可协商 |
| 必须修正 | 提示词结构或规范错误 | 修正后才能执行 |
| 建议优化 | 提示词质量可提升的点 | 建议处理 |
