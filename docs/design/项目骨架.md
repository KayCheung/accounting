# 项目骨架

**你当前所在目录是 accounting/，请严格按照这个目录生成代码**

```
accounting
├── docs
│   ├── design
│   │   ├── flowchart               // 核心流程图
│   │   ├── images                  // png类型的流程图，无需关注
│   │   └── prototypes              // 原型
│   ├── prompt                      // 提示词
│   └── sql                         // 表结构 DDL SQL
├── accounting-api                  // deploy到Maven仓库，提供给需要的服务直接依赖
│	└── src/main/java/com/kltb/accounting
│       ├── feign                   // 对外提供feign接口
│       ├── enums                   // 符合 Mp 映射规范的枚举类
│       ├── constants               // 常量类
│       └── model                   // 输入输出模型对象(request、response)
└── accounting-core	            // 核心业务实现
	├── src/main/java/com/kltb/accounting
	│   ├── controller              // 接口层：RESTful API 暴露
	│   ├── service                 // 业务编排层：负责事务组合与编程式事务管理
	│   ├── domain                  // 领域核心层
	│   │   ├── dto                 // 领域对象与聚合根、业务之间传输
	│   │   ├── strategy            // 记账策略实现 (多态、模板方法)
	│   │   └── service             // 领域服务：核心算法、余额计算
	│   ├── repository              // 持久化层
	│   │   ├── mapper              // MyBatis-Plus Mapper 接口
	│   │   └── entity              // 数据库映射对象 (DO)
	│   └── infrastructure          // 基础设施层
	│       ├── config              // 配置类 (Redisson, Nacos, Mp)
	│       ├── exception           // 全局异常处理与 ResultCode
	│       ├── util                // 精度保障工具类 (MoneyUtil)
	│       └── metrics             // Prometheus 监控埋点
	└── src/test/java/com/kltb/accounting  // 基于 JUnit 5 + AssertJ 的单元测试
```

**【强制原则】**：
   - accounting-api 模块严禁引入 mybatis-plus、jdbc 等持久层依赖。
   - accounting-core 必须通过 Maven 依赖引用 accounting-api 中的枚举和模型。
   - 所有生成的 Java 文件必须严格遵守上述对应的包（Package）路径。
