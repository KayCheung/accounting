# step-01-governance

> 本文件为Step 1开发契约与规范的详细文档，明确任务目标、执行标准及后续行动。

## 1. 任务目标（Mission）
制定项目开发核心契约与规范，明确AI三层规则、VC上手标准，完成Prompt目录初始化及全员对齐，为后续开发奠定统一基础。

## 2. 详细子任务列表（含技术要点）

### 2.1 TL：制定 Rules 三层规则文件

- `docs/ai-rules/general.md`：角色定义、技术栈、AI输出格式约束。

- `docs/ai-rules/java.md`：分层架构、异常处理、事务边界、BigDecimal强制规范。

- `docs/ai-rules/accounting.md`：幂等键、借贷平衡、锁顺序、领域术语速查。

### 2.2 TL：制作《Vibe Coding 一页纸快速上手》

- Prompt四段式模板（任务/输入/输出/不要做）。

- 会话拆分、上下文管理规则，提升交互效率。

- 要求AI先复述任务再执行，避免理解偏差。

### 2.3 VC：初始化 `docs/prompt/` 目录，创建全部Step占位文件

- 按规范创建目录，确保结构合规。

- 创建所有Step占位文件，保持格式统一。

### 2.4 全员：30分钟对齐会 + VC完成首个Prompt练习

- TL主持对齐会，明确核心约束，解答疑问。

- VC按模板完成首个Prompt练习，掌握规范。

## 3. 完成标准（Checklist）

- ✅ `docs/ai-rules/` 下三个规则文件就位且完整。

- ✅ 《Vibe Coding 一页纸快速上手》制作完成。

- ✅ `docs/prompt/` 目录及所有Step占位文件创建完成。

- ✅ 全员对齐会完成，核心约束确认无异议。

- ✅ VC完成首个Prompt练习，符合规范。

## 4. 下一步行动

1. TL审核VC的Prompt练习，指导修正。

2. VC同步上手文档及规则文件至团队共享渠道。

3. 进入Step 2，全员遵循规范执行，TL定期检查。

4. TL牵头优化规则文件中的漏洞及不合理之处。
