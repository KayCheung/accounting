# Task 6: Buffer Posting & Batch Processor
1. **BufferEntryService**:
   - 当 `t_voucher_rule` 匹配到异步模式时，由记账引擎调用。
   - 逻辑：根据 `t_buffer_posting_rule` 将分录写入 `t_buffer_posting_detail`。
   - 状态初始化为 '待处理'。
2. **BufferPostingJob (后台任务)**：
   - 扫描 `t_buffer_posting_detail` 中状态为 '待处理' 或 '失败' 且重试次数 < 3 的记录。
   - **批量入账逻辑**：
     - 按 `account_no` 分组，对同一账户的金额进行汇总。
     - 使用 **乐观锁 (version)** 更新 `t_account` 余额。
     - 成功后，批量更新明细状态为 '已处理'，记录 `complete_time`。
     - 失败则记录 `fail_reason` 并增加 `retry_count`。
3. **要求**：
   - 必须处理 `actual_accounting_date`，确保即便跨日处理，会计日期仍保持原业务日期。
   - 汇总入账必须记录一笔汇总性质的 `t_account_detail`。
