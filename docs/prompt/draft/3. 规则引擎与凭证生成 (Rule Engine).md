# Task 3: Accounting Rule Engine
实现 `RuleEngineService`，逻辑如下：
1. **输入**：`BusinessRecord` 和 `List<BusinessDetail>`。
2. **处理**：
   - 匹配 `t_voucher_rule` 中的 `rule_script`。
   - 遍历 `t_voucher_rule_detail`。
   - **方案 A/B 兼容**：如果规则明细有固定 `aux_code` 则使用方案 A；如果有 `rule_script` 则使用 SpEL 从业务明细的 `extra_json` 中动态提取（方案 B）。
3. **输出**：生成 `AccountingVoucher` 及其关联的 `VoucherEntryDetail` 列表。
4. **校验**：生成的借贷分录金额必须合计相等（或净额为 0）。
