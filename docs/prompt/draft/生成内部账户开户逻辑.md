# Task 1: Internal account opening processing logic

## 背景
- 会计科目`t_account_subject` 中：
  - `allow_post = 1`（允许记账）
  - `allow_open_account = 0`（不允许为客户开户）
- 账户主表 `t_account` 中：
  - 内部账户标识：`ownerType = "INNER"` 且 `ownerId = "INNER"`
  - 账户号唯一约束：`accountNo`，账号生成规则调用 `InnerAccountNoGenerator.generate` 方法生成
  - 账户名称：`accountName`，名称生成规则调用 `InnerAccountNameGenerator.generate` 方法生成

这类科目**不能通过客户开户流程创建账户**，但**必须存在对应的内部账户**，以便凭证分录能正确写入 `t_account` 和 `t_account_detail` 表。

### 需求
在已有 `AccountOpenService` 类中，**新增方法 `void createInnerAccount(AccountSubject subject)`**，实现以下逻辑：
1. **校验参数subject是否满足开内部账户的条件**：
   - `allowPost = true`
   - `allowOpenAccount = false`
   - `isDelete = false`
   - `is_leaf = true`
   - `status = 1`

2. **检查是否已存在对应的内部账户**：
   - 账户号格式：`"INTERNAL_" + subjectCode`（如 `"INTERNAL_6001"`）
   - 判断依据：`accountNo` 唯一

3. **若不存在，则创建内部账户**，设置以下属性：
   - `subjectCode`：科目编码
   - `accountNo`：`"INTERNAL_" + subjectCode`
   - `accountName`：`科目名称 + "（内部汇总户）"`
   - `ownerType`：`"内部账户"`
   - `ownerId`：`"INNER"`
   - `currency`：`"CNY"`
   - `balanceDirection`：取自科目的 `direction`
   - `status`：`"正常"`
   - `riskStatus`：`"正常"`
   - `openDate`：当前日期
   - 其他字段使用合理默认值（如 `balance = 0`, `version = 0`）

4. **幂等性**：已存在的账户不重复创建。
5. **事务性**：整个方法需在事务中执行。
6. **日志**：记录成功创建的账户数量及明细（INFO 级别）。

### 输出要求
仅生成 `AccountOpenService` 类中的 `createInnerAccount()` 方法完整代码，包含必要注解（如 `@Transactional`）和异常处理。





## 背景
在账务系统中，部分会计科目（`t_account_subject`）设置为：
- `allow_post = 1`（允许记账）
- `allow_open_account = 0`（不允许为客户开户）

这类科目**不能通过客户开户流程创建账户**，但**必须存在对应的内部账户**，以便凭证分录能正确写入 `t_account` 和 `t_account_detail` 表。

## 要求
1. 编写一个 **Spring Boot 3.x + JPA** 的服务方法，用于**系统启动时自动创建缺失的内部账户**。
2. 内部账户特征：
   - `owner_type = '内部账户'`
   - `owner_id = 'INNER'`
   - `account_no` 格式：`INTERNAL_{subject_code}`（如 `INTERNAL_6001`）
   - `account_name = 科目名称 + "（内部汇总户）"`
   - `status = '正常'`
   - 其他字段使用合理默认值（如 currency='CNY', balance_direction 根据科目 direction 设置）
3. **幂等执行**：已存在的内部账户不再重复创建。
4. 使用 `@PostConstruct` 或 `ApplicationRunner` 触发。
5. 日志记录创建结果（SLF4J）。

## 输入表结构参考
- `t_account_subject`: 包含 `subject_code`, `subject_name`, `direction`, `allow_post`, `allow_open_account`
- `t_account`: 包含 `account_no`, `subject_code`, `owner_type`, `owner_id`, `account_name`, `balance_direction` 等
