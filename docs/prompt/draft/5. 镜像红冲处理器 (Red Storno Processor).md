# Task 5: Red Storno Processor
实现 `StornoService`，要求：
1. **输入**：`orig_trace_no`。
2. **逻辑**：
   - 查找原凭证 `t_accounting_voucher` 及其所有分录。
   - **镜像生成**：方向（借/贷）与原分录完全一致，金额 = 原金额 * -1。
   - 凭证头 `orig_voucher_no` 指向原凭证，`trans_type` 设为 '红'。
3. **联动**：如果原交易涉及解冻扣款，红冲时需自动触发“反向冻结”，将金额压回 `t_sub_account` 的冻结余额。
4. **事务**：所有操作必须在同一个事务内完成。
