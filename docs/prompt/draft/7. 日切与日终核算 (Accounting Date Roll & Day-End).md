# Task 7: Day-End Processing & Reconciliation
1. **AccountingDateManager (日切逻辑)**：
   - 实现日切接口：更新系统当前的会计日期。
   - 逻辑：检查当天所有“实时入账”事务是否已关闭，缓冲入账是否已全部处理完成（或达到重试上限）。
2. **DayEndReconciler (日终核算)**：
   - **借贷平衡检查**：汇总当日 `t_voucher_entry_detail`，验证 $\sum \text{Debit} = \sum \text{Credit}$。
   - **总分平衡检查**：
     - 计算 `t_account` 的当日变动额（借方合计 - 贷方合计）。
     - 验证 `t_account.balance` (期末) = `opening_balance` (期初) + 变动额。
   - **生成日余额表**：
     - 将核算结果写入 `t_account_balance`。
     - 如果发现不平，记录差额并触发 `SystemAlarm` 告警。
3. **SnapshotService (余额快照)**：
   - 日切完成后，对所有 `t_account` 执行 `t_account_balance_snapshot` 的批量插入，保留历史水位。
