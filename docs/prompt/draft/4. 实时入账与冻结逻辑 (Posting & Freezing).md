# Task 4: Transactional Posting & Freezing Service
1. **RealTimePostingService**:
   - 必须开启数据库事务。
   - 使用 `FOR UPDATE` 锁定 `t_account` 和 `t_sub_account`。
   - 调用 `BalanceCalculator` 获取 `post_balance`。
   - 同步写入 `t_account_detail` (明细账) 和更新账户余额。
2. **FreezingService**:
   - 实现独立的同步冻结/解冻接口。
   - 逻辑：可用余额与冻结余额之间的平移。
   - 写入 `t_sub_account_detail`。
   - 必须处理 `orig_trace_no` 实现幂等，防止重复冻结。
