# Task 8: Transaction Compensation Job
实现 `TransactionRecoveryJob`，用于处理异常中断的入账流程：
1. **扫描孤儿事务**：
   - 扫描 `t_transaction` 中状态为 '未提交' 且创建时间超过 5 分钟的记录。
2. **状态补偿逻辑**：
   - 根据 `trace_no` 到 `t_accounting_voucher` 和 `t_account_detail` 中查询是否已存在成功记录。
   - **若已存在部分记录**：说明更新中途崩溃，尝试“向前推进”，补齐缺失的余额更新或凭证记录。
   - **若均不存在**：说明事务根本未开始执行，根据业务规则决定是标记为 '已回滚' 还是重新发起入账。
3. **幂等性保障**：
   - 补偿过程中必须严格校验 `uk_trace_no` 和 `uk_entry_id`，防止重复入账。
